# カロリー管理アプリ
自分のプロフィール情報から理想の摂取カロリーを求め、1日に摂取したカロリーとの差を管理するWebアプリケーションです。

# 開発した背景
プログラミングを学び、何か自分の興味ある事柄をアプリにしたいと考え、簡易的なToDoアプリの次にこのアプリの開発に取り組みました。
そこで、近年の筋トレブームや健康への意識向上に伴い、自分に必要な摂取カロリーを求めて可視化し、毎日の食事を記録することで健康管理の手助けになるのではないかと考えています。
また、競技プログラミングからプログラミング学習を始めたこともあり、計算する要素を取り入れたいと考えていたことも一つの理由です。

# 使用技術
  - Java 21
  - Spring Boot 4.0.5
  - Spring Security
  - React ← (Tymeleaf)
  - CSS
  - MySQL 

# 機能一覧
　- ユーザー登録、ログイン
　- 食事のカロリー記録、削除
  - 過去の一日の食べたもの検索、削除

# 開発の経緯
プログラミング学習未経験で、Javaでの競技プログラミングの学習を軸に始めたため、Webアプリケーションがどのようにして動いているのかということ自体知りませんでした。そこで、25年12月頃より、UdemyにてWebアプリケーション初心者向けの講座にて学習を始めました。HTMLやCSSなど、主にどのように表示されるかを学びました。

次に、自分がJavaを軸に学習を進めていたため、Sprng Bootの学習を始めました。この学習をもとに、今回のカロリー管理アプリの作成を始めました。
しかし、先に機能の部分を実装して取り組むべきだったところ、アプリとして利用するのであればログイン機能が必要であると考え、並行してSpring Securityの勉強を始めました。これによって、アプリの機能の部分とログイン部分とでどのように順序だてて取り組むかといった点や、知識として浅いため何度も授業を見返すといった点が重なり、非常に時間をかけてしまい効率が悪かったと反省しています。

次に、現在はSPAが主流ということを知り、フロントエンドにReact、バックエンドにSpringという構成にしようと考えました。(Reactを選択した理由は、JSを少し学び、Udemyで講座が安くなっていたため)それぞれ機能を実装したが、React-Spring間でデータをやり取りする方法が分からず、Reactのapi.jsとSpringのSecurityConfigクラスは、生成AIを利用しました。

今後の展望として、機能面ではタンパク質や脂質、そして炭水化物など項目ごとにも記録をつけていきたい。また、DockerやAWS等の勉強もして構築してデプロイまでもっていきたい。そして、UIの面がまだ未熟なのでより自然なUIにできるように。
